<task type="auto" tdd="true">
  <name>Task 2: Git worktree lifecycle functions (TDD)</name>
  <files>autopilot/src/worker/git-worktree.ts, autopilot/src/worker/__tests__/git-worktree.test.ts</files>
  <behavior>
    - createWorktree(projectDir, phaseNumber) creates worktree at ../{repo}-worktrees/phase-{N}/ on branch gsd/phase-{N}, returns the worktree path
    - mergeWorktree(projectDir, phaseNumber) runs `git merge gsd/phase-{N} --no-edit` from projectDir, returns true on success, false on conflict
    - cleanupWorktree(projectDir, phaseNumber) removes the worktree and deletes the branch
    - ensureCleanWorktree(projectDir, phaseNumber) cleans up stale worktree/branch before creating fresh, idempotent (no error if nothing to clean)
    - execGit(cwd, args) is a promisified wrapper around child_process.execFile for git commands
    - All functions handle Windows paths correctly (use execFile, not exec)
  </behavior>
  <action>
TDD: Write tests first in `autopilot/src/worker/__tests__/git-worktree.test.ts`, then implement in `autopilot/src/worker/git-worktree.ts`.

Tests should create a temporary git repo (using `tmp` from vitest or `os.tmpdir`), exercise the full lifecycle: ensureClean -> create -> verify branch exists -> merge -> cleanup -> verify branch and worktree gone.

Implementation uses `node:child_process` `execFile` (NOT `exec`) per established project pattern. Path construction uses `node:path` `basename`, `resolve`, `join`.
