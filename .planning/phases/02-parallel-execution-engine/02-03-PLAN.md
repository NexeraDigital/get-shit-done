    - On worker completion in parallel mode: merge worktree, on success cleanup worktree, on merge failure mark result.mergeSuccess=false and preserve worktree

Write tests first in `autopilot/src/worker/__tests__/worker-pool.test.ts`, then implement `autopilot/src/worker/index.ts`.

WorkerPool extends EventEmitter. Key implementation details:

1. **Constructor:** Store options. Create a merge queue (promise chain for serialized merges).

2. **dispatch(phase, runPhaseFn):**
   - `runPhaseFn` is `(cwd: string, claudeService: ClaudeService) => Promise<void>` -- the orchestrator passes its existing `runPhase` logic here. This avoids the WorkerPool needing to know about PhaseState internals.
   - In parallel mode: `ensureCleanWorktree` -> `createWorktree` -> create `new ClaudeService({ defaultCwd: worktreePath })` -> call `runPhaseFn(worktreePath, claudeService)` -> on completion, enqueue serialized merge -> on merge success, cleanup worktree.
   - In sequential mode: `runPhaseFn(this.projectDir, claudeService)` directly with no worktree.
   - Store WorkerHandle in `this.active` Map keyed by phaseNumber.
   - Wire ClaudeService events to re-emit with `{ ...event, phaseNumber }`.

3. **waitForAny():** `Promise.race` over all active worker promises. Remove completed worker from active Map. Return WorkerResult.

4. **activeCount:** getter returning `this.active.size`.

5. **abortAll():** Call `claudeService.abortCurrent()` on all active workers. Used by ShutdownManager.

For tests: mock ClaudeService (just needs to be an EventEmitter with runGsdCommand and abortCurrent). Mock git-worktree functions. Verify dispatch creates workers, waitForAny resolves on completion, merge is called in parallel mode, cleanup happens after successful merge.

Per RESEARCH pitfall 4: The IPC paths (events, heartbeat, state) must use the MAIN repo's projectDir, not the worktree path. The ClaudeService cwd is the worktree, but the orchestrator-level components (EventWriter, StateStore) are not created by WorkerPool -- they remain at orchestrator level pointed at the main repo. **Changes to Orchestrator:**

1. **Add parallel/concurrency to run() signature:**

2. **Bridge RoadmapPhase[] to SchedulerPhase[]:**
   After `extractPhases()`, convert using `parseDependsOn`:
   ```typescript
   import { DependencyScheduler, type SchedulerPhase } from '../scheduler/index.js';
   import { parseDependsOn } from '../scheduler/parse-depends-on.js';

   function toSchedulerPhases(roadmapPhases: RoadmapPhase[]): SchedulerPhase[] {
     return roadmapPhases.map(rp => ({
       number: rp.number,
       name: rp.name,
       dependencies: parseDependsOn(rp.dependsOn),
     }));
   }
   ```

3. **Replace the phase loop (lines 374-386)** with:
   ```typescript
   const parallel = options?.parallel ?? false;
   const concurrency = parallel ? (options?.concurrency ?? 3) : 1;

   // Filter to applicable phases (not completed, in range)
   const applicablePhases = currentState.phases
     .filter(p => p.status !== 'completed' && p.status !== 'skipped')
     .filter(p => !phaseRange || phaseRange.includes(p.number));

   // Build scheduler from roadmap dependencies
   const schedulerPhases = toSchedulerPhases(
     applicablePhases.map(p => ({
       number: p.number,
       name: p.name,
       completed: false,
       inserted: p.inserted ?? false,
       dependsOn: p.dependsOn ?? null,
     }))
   );
   const scheduler = new DependencyScheduler(schedulerPhases);

   // Create worker pool
   const workerPool = new WorkerPool({ concurrency, parallel, projectDir: this.projectDir });

   // Register worktree cleanup with ShutdownManager (if available)
   // WorkerPool.abortAll() should be callable from shutdown handler

   // Unified loop
   while (!scheduler.isComplete()) {
     if (this.shutdownRequested) break;

     const ready = scheduler.getReady();
     for (const readyPhase of ready) {
       if (workerPool.activeCount >= concurrency) break;
       scheduler.markInProgress(readyPhase.number);

       const phaseState = currentState.phases.find(p => p.number === readyPhase.number)!;
       workerPool.dispatch(readyPhase, async (cwd, claudeService) => {
         // In parallel mode, each worker gets its own ClaudeService
         // Store original and temporarily swap for this phase
         await this.runPhase(phaseState);
       });
     }

     // Wait for at least one worker to complete
     if (workerPool.activeCount > 0) {
       const result = await workerPool.waitForAny();
       if (result.success) {
         const newlyReady = scheduler.markComplete(result.phaseNumber);
         // newlyReady phases will be picked up on the next loop iteration
       } else {
         // Phase failed -- in default mode, request shutdown
         // (--continue mode is Phase 3 scope)
         this.logger.log('error', 'orchestrator',
           `Phase ${result.phaseNumber} failed: ${result.error}`);
         if (!result.mergeSuccess && result.mergeSuccess !== undefined) {
           this.logger.log('warn', 'orchestrator',
             `Merge conflict for phase ${result.phaseNumber} -- worktree preserved`);
         }
         this.requestShutdown();
       }
     }
   }
   ```

This keeps backward compatibility -- sequential mode still uses the shared instance.
