  <behavior>
    - StateWriteQueue serializes concurrent setState() calls -- no interleaved reads/writes
    - When 3 concurrent enqueue() calls modify different fields, all 3 fields are present in final state (no lost updates)
    - StateWriteQueue.enqueue() returns a promise that resolves after the operation completes
    - If an enqueued operation throws, subsequent operations still execute (fault isolation)
    - StateStore can be used with or without the queue (queue is an external wrapper, not baked into StateStore)
  </behavior>
  <action>
    1. Add `StateWriteQueue` class to `autopilot/src/state/index.ts`:
       ```typescript
       export class StateWriteQueue {
         private chain: Promise<void> = Promise.resolve();

         enqueue(fn: () => Promise<void>): Promise<void> {
           const next = this.chain.then(fn, fn);
           this.chain = next;
           return next;
         }
       }
       ```
       This is the simple promise-chain queue from RESEARCH.md. No external library needed -- Node.js is single-threaded; this serializes async operations that interleave at await points.
