  <behavior>
    - markFailed(phaseNumber) marks a phase as failed, removes from inProgress, and transitively marks all dependent phases as skipped
    - markFailed returns an array of SchedulerPhase[] that were skipped (the transitive dependents)
    - markSkipped(phaseNumber) marks a phase as skipped (used internally by markFailed for dependents)
    - isComplete() returns true when completed.size + failed.size + skipped.size === phases.size

  <files>autopilot/src/worker/merge-resolver.ts, autopilot/src/worker/__tests__/merge-resolver.test.ts, autopilot/src/worker/types.ts, autopilot/src/worker/git-worktree.ts</files>
  <behavior>
    - resolveConflicts(projectDir, phaseNumber, priorReports) detects conflicting files via `git diff --name-only --diff-filter=U`
    - For each conflicting file, resolves using `git checkout --theirs -- <file>` then `git add <file>`
    - After all files resolved, runs `git merge --continue --no-edit` to complete the merge
    - If resolution fails partway, calls `git merge --abort` to clean up merge state
    - Returns a MergeReport with: phaseNumber, files (with file path, strategy, outcome), timestamp, success boolean
    - writeMergeReport(phaseDir, report) writes structured markdown to `{phaseDir}/merge-report.md`
    - Report markdown includes: header, phase number, timestamp, table of files with strategy and outcome
    - MergeReport type exported from merge-resolver.ts (not types.ts -- keeps it co-located)
    - WorkerResult extended with optional mergeReport field (MergeReport | undefined)
    - priorReports parameter is passed through but used only for context documentation in the report (which prior strategies were used)
  </behavior>
