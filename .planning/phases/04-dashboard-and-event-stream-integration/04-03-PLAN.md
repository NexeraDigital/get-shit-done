<task type="auto">
  <name>Task 1: Enhance PhaseCard with parallel status and question badges</name>
  <files>autopilot/dashboard/src/components/PhaseCard.tsx</files>
  <action>
    Modify the PhaseRow component in PhaseCard.tsx to show parallel execution awareness:

    1. **Worker status badge:** When `phase.workerStatus` is defined, show a small text badge next to the step dots indicating the parallel worker state. Use these styles per locked decision (instant updates, no animations):
       - `running`: blue background, "running" text
       - `queued`: gray background, "queued" text (grayed out with dependency info per decision)
       - `done`: green background, "done" text
       - `failed`: red background, "failed" text

    2. **Multiple active phases:** The existing `isActive` logic uses `isCurrent` which only matches one phase. For parallel mode, change the `isActive` check in StepDot to: `isActive = stepStatus !== 'idle' && stepStatus !== 'done' && stepStatus === step` (remove the `isCurrent` guard). This allows multiple phases to show blue-pulse dots simultaneously. Keep the `isCurrent` prop for the border highlight but base it on `phase.workerStatus === 'running'` when workerStatus is defined, falling back to `phase.number === currentPhase` for sequential mode.

    3. **Per-phase question badge:** Accept a `questionsByPhase` prop (Map<number, number> mapping phase number to pending question count). In PhaseRow, if `questionsByPhase.get(phase.number)` is > 0, show a small amber badge with the count next to the phase name. Example: "Phase 2: Parallel Engine [2]" where [2] is the question count badge.

    4. **Failed phase error display (discretion: badge + expandable inline):** When `phase.workerStatus === 'failed'` and `phase.error` is defined, show a collapsible error section below the step dots. Initially collapsed, click to expand. Use red-50 background with red-700 text.

    Update the PhaseCard component to compute `questionsByPhase` from the questions prop. Add `questions` to PhaseCardProps:
    ```typescript
    interface PhaseCardProps {
      phases: PhaseState[];
      currentPhase: number;
      questions?: QuestionEvent[];  // NEW
    }
    ```
    Compute the map inside PhaseCard:
    ```typescript
    const questionsByPhase = new Map<number, number>();
    for (const q of (questions ?? [])) {
      const phase = q.phase ?? 0;
      questionsByPhase.set(phase, (questionsByPhase.get(phase) ?? 0) + 1);
    }
    ```
  </action>

  </verify>
